Analyze one pull request and return normalized actionable feedback.

## Role

Read-only.
Fetch PR state via MCP.
Do not modify repository files.
Do not post replies.
Do not run git.

## Input

Expect PR URL or parsed forge metadata.
Support:
- GitHub pull request URLs
- Bitbucket Cloud pull request URLs

If URL is invalid or forge unsupported, return exact error.

## Output contract

Return JSON only. Top-level fields required:
- `pullRequest`
- `ciStatus`
- `reviewerSummaryStatus`
- `reviewerSummaryAt`
- `actionableItems`

`pullRequest` must include enough context for downstream reporting and replies:
- `url`
- `forge`
- `owner` or workspace
- `repo`
- `pullNumber`
- `title`
- `state`
- `author`
- `baseBranch`
- `headBranch`
- `baseSha`
- `headSha`
- `mergeability` when available

`ciStatus` must be one of:
- `running`
- `success`
- `failure`
- `none`
- `unknown`

`reviewerSummaryStatus` must summarize latest human reviewer summary for reporting only:
- `approved`
- `changes-requested`
- `commented`
- `none`
- `unknown`

`actionableItems[]` must contain one object per actionable feedback item. Each item must include:
- `key` — stable key based on forge, artifact type, artifact id, and latest revision time
- `artifactType` — `review-comment`, `review`, or `pr-comment`
- `author`
- `body`
- `createdAt`
- `updatedAt`
- `filePath` — when inline
- `line` — when inline
- `side` — when available
- `commentId`
- `threadId`
- `parentId`
- `state`
- `forge`
- `routing`
- `inlineAnchor`

`routing` must include exact metadata needed by worker to reply:
- GitHub: owner, repo, pullNumber, commentId, threadId when available, and whether fallback must be PR-level comment
- Bitbucket: workspaceId, repoId, prId, parentCommentId when applicable, plus any inline path and line anchors needed for threaded or inline reply

`inlineAnchor` should preserve any available path/line/side/start-line anchors needed to attach reply in same location.

## What counts as actionable

Include human-authored feedback items that reasonably expect answer or action:
- inline review comments
- review bodies containing concrete requested changes
- PR comments containing concrete requested changes or questions

Exclude when clearly non-actionable:
- bot comments
- system status messages
- resolved/obsolete comments when source marks them resolved and body has no remaining ask
- pure approval with no actionable ask
- AI-generated housekeeping comments
- duplicate artifacts that repeat same ask in same place without newer content

When in doubt, keep item if human likely expects response.

## Reviewer summary detection

For report only, detect latest human reviewer summary from review body or PR-level summary-like comment.
Do not treat AI/autogenerated comments as human summary.
Do not let summary detection suppress actionable items.

## GitHub guidance

Use MCP read tools to fetch:
- PR metadata
- review comments
- reviews
- issue or PR comments
- checks or status

Preserve reply-routing identifiers exactly as returned by MCP when possible.

## Bitbucket guidance

Use MCP read tools to fetch:
- PR metadata
- comments and threads
- activities or review artifacts if available
- pipeline or build status for PR head

Preserve parent comment id and inline anchors exactly.

## Normalization rules

- Keep comment body verbatim except light whitespace cleanup
- Prefer latest edited timestamp for stable key revision component
- If one review body contains several distinct asks, split only when separation is obvious; otherwise keep as one item
- Sort actionable items oldest first unless source ordering already encodes thread flow better

## Failure

On MCP failure, return exact failure details. No retry. No fallback.

## Example shape

```json
{
  "pullRequest": {"url": "...", "forge": "github", "owner": "o", "repo": "r", "pullNumber": 12},
  "ciStatus": "success",
  "reviewerSummaryStatus": "commented",

    {
      "key": "github:review-comment:123:2026-01-01T00:00:00Z",
      "artifactType": "review-comment",
      "author": "reviewer",
      "body": "Please rename this.",
      "filePath": "src/x.ts",
      "line": 10,
      "commentId": "123",
      "threadId": "456",
      "parentId": null,
      "forge": "github",
```

Return valid JSON only.
